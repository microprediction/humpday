Tabu Search (Custom)

Memory-Based Local Search Metaheuristic

← Back to Optimizer Competition
Tabu Search is a sophisticated local search metaheuristic that uses memory structures to guide the optimization process. Developed by Fred Glover in the 1980s, it overcomes local optima by maintaining a "tabu list" of recently visited solutions or moves, forcing the algorithm to explore new areas even when they temporarily worsen the objective function.

Memory-Guided Search

Tabu Search uses intelligent memory to guide exploration:

  • Tabu List: Remembers recent moves to avoid cycling
  • Forbidden Moves: Prevents returning to recently visited solutions
  • ⭐ Aspiration Criteria: Override tabu status for exceptional solutions
  • Strategic Oscillation: Systematically explore around best-found solutions

  • Function evaluations: Moderate, depends on neighborhood size
  • Convergence: Good escape from local optima, strategic exploration
  • Robustness: Memory prevents cycling, handles complex constraint landscapes

Core Components

  • Current Solution: s ∈ S (current position in solution space)
  • Neighborhood: N(s) = {s' | s' is reachable from s in one move}
  • Tabu List: T = {forbidden moves or solutions}
  • Tabu Tenure: Duration moves remain forbidden
  • Aspiration Criteria: Conditions to override tabu restrictions
  • Best Solution: s* = best solution found so far

Algorithm Structure

  • Initialization: Start with initial solution s₀, empty tabu list T = ∅
  • Neighborhood Generation: Create candidate moves N(s) \ T
  • Move Selection: Choose best non-tabu move (or aspiration override)
  • Tabu List Update: Add reverse move to T, remove expired entries
  • Solution Update: Move to selected neighbor, update best if improved
  • Termination: Stop based on iterations, time, or convergence

️ Key Design Decisions

  • Tabu Tenure: Fixed (5-10) vs adaptive vs random
  • Tabu Structure: Moves vs solutions vs attributes
  • Aspiration Criteria: Best solution, threshold, or frequency-based
  • Neighborhood: Size vs quality trade-off
  • Diversification: Long-term memory for exploration
  • Intensification: Return to promising regions

Tabu Search Pseudocode

function tabuSearch(initialSolution, maxIterations):
    s = initialSolution
    bestSolution = s
    tabuList = []

    for iter in 1 to maxIterations:
        neighborhood = generateNeighbors(s)
        candidateList = []

        for each neighbor in neighborhood:
            move = getMoveFromTo(s, neighbor)

            if not isTabu(move, tabuList) or
               satisfiesAspiration(neighbor, bestSolution):
                candidateList.add(neighbor, move)

        if candidateList.isEmpty():
            break  // No valid moves

        // Select best candidate
        (nextSolution, bestMove) = selectBest(candidateList)

        // Update tabu list
        tabuList.add(reverseMove(bestMove), currentIteration)
        removeExpiredMoves(tabuList, currentIteration)

        // Move to next solution
        s = nextSolution
        if f(s) < f(bestSolution):
            bestSolution = s

    return bestSolution

Tabu Search Variants

  • Simple Tabu Search: Basic tabu list with fixed tenure
  • Reactive Tabu Search: Adaptive tabu tenure based on search history
  • Robust Tabu Search: Multiple tabu lists and escape mechanisms
  • Parallel Tabu Search: Multiple concurrent searches
  • Hybrid Tabu Search: Combined with other metaheuristics
  • Granular Tabu Search: Restrict neighborhoods for large problems

Memory Types

  • Short-Term (Recency): Tabu list preventing recent moves
  • Medium-Term (Frequency): Track frequently used moves/attributes
  • Long-Term (Diversification): Guide search to unexplored regions
  • Strategic (Intensification): Return to promising solution regions

Applications

  • Vehicle Routing: TSP, VRP, and delivery optimization
  • Scheduling: Job shop, flow shop, and timetabling
  • Facility Location: Warehouse and service center placement
  • Graph Coloring: Frequency assignment and conflict resolution
  • Portfolio Selection: Financial optimization with constraints
  • VLSI Design: Circuit placement and routing

Tabu Search Philosophy

  • Aggressive Exploration: Accept worse solutions to escape local optima
  • Intelligent Memory: Use history to guide future decisions
  • Strategic Oscillation: Balance intensification and diversification
  • Flexible Framework: Adapt to problem-specific structures
  • No Randomness Required: Deterministic yet effective exploration

️ Implementation Challenges

  • Neighborhood Design: Balance size vs computation time
  • Tabu Tenure: Too short (cycling) vs too long (over-restriction)
  • Memory Management: Efficient data structures for large problems
  • Parameter Tuning: Problem-dependent parameter sensitivity
  • Aspiration Criteria: Balancing exploration vs exploitation
